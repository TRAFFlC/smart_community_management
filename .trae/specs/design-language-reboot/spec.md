# 前端设计语言重构 Spec

## Why

当前前端呈现典型的「AI 生成式中后台」观感：圆角小卡片、 emoji 与细线图标混用、经典商务蓝主题、表格与正文字体缺乏层级。这种设计难以建立专业感与长期使用意愿。本次重构不以「换色」为目标，而是重新建立一套有统一叙事的设计语言：更像「社区运营指挥中心」或「城市档案室」，强调信息密度、字体层级、清晰的模块边界，以及克制但有力的强调色。

## What Changes

### A. 建立新的设计系统（Design Tokens）
- 重新定义主色、强调色、中性色、语义色，替换现有 `#1677ff` 主导商务蓝
- 定义字体层级（标题/正文/辅助/数据），统一字重与行高
- 定义间距系统（4px 基网 → 8/12/16/24/32/48）
- 定义圆角策略：卡片/面板 4px 微圆角，按钮/标签 4px，彻底弱化「小圆角卡片」印象
- 定义阴影与边框：以 1px 实色细边框为主，仅在悬浮/聚焦时使用极淡投影

### B. 全局框架重构
- 侧边栏：深色窄栏（72px 图标栏 + 展开面板）或扁平分组，减少子模块堆砌感
- 顶栏：降低高度，去掉多余边框，搜索框与工具按钮使用新样式
- 面包屑：改为更克制的文本路径，可点击项带下划线 hover
- 页面容器：统一边距与背景色，消除各页面「三段式」同质布局

### C. 工作台（Dashboard）重新设计
- 移除四张等宽统计卡片的堆砌，改为「今日关键数字」大字体展示区
- 快捷操作改为图标网格，配合文字标签，不再使用带边框小卡片
- 待办事项使用紧凑列表，去掉 emoji，使用语义色圆点
- 社区动态与时间线合并，使用左侧色条区分事件类型

### D. 表格与列表重构
- 表头：使用浅灰背景 + 深灰文字 + 11px 大写/小字号标签化样式，增强与数据行的区分
- 数据行：统一 44px 行高，增加斑马纹，hover 使用整行背景变化而非仅文字变色
- 状态标签：使用小写/短文本 + 4px 圆角 + 实色背景（低饱和），不使用大圆角
- 操作列：改为 icon-only 按钮组或文字链接，去除胶囊按钮
- 分页条：整合到底部，使用更紧凑的样式

### E. 图标与图形
- 移除所有 UI 中的 emoji（如 📋 / 📌 / 📢）
- 统一使用 SVG 图标，规范尺寸（16px 辅助 / 20px 常规 / 24px 导航）
- 图标风格统一为 2px 线宽、无填充的「工具图标」风格
- 统计图表配色同步到新设计系统

### F. 表单与弹窗
- 输入框：降低圆角至 4px，聚焦时使用强调色边框
- 按钮：主按钮使用新的强调色；次级按钮使用白底细边框
- 弹窗：统一标题字号与内边距，减少多余分隔线

## Impact

- Affected specs: platform-ux-overhaul（样式覆盖）, functional_spec.md（界面描述）
- Affected code:
  - `resources/style.qss`：全局样式重写
  - `src/widgets/MainWindow.cpp`：框架、侧边栏、顶栏、工作台、各页面样式调整
  - `src/widgets/LoginWidget.cpp`：登录页视觉同步
  - `resources/icons/*.svg`：如资源允许，逐步替换为统一风格图标

## ADDED Requirements

### Requirement: 新的设计系统
The system SHALL 使用一套统一的设计 token，包括颜色、字体、间距、圆角、阴影。

#### Scenario: 全局样式一致性
- **WHEN** 开发者查看 `resources/style.qss`
- **THEN** 所有颜色与尺寸均以 token 注释形式集中定义，而不是分散的硬编码十六进制值
- **AND** 任意两个页面在相同语义元素上使用相同样式

### Requirement: 移除 emoji
The system SHALL 不在任何 UI 文本中使用 emoji 字符。

#### Scenario: 工作台待办列表
- **WHEN** 用户打开工作台
- **THEN** 待办项、公告、快捷操作等位置仅显示 SVG 图标或色块
- **AND** 不再出现 📋 / 📌 / 📢 等字符

### Requirement: 深色侧边栏与浅色内容区
The system SHALL 使用深色侧边栏与浅色内容区形成清晰的结构分区。

#### Scenario: 默认主题
- **WHEN** 用户登录后
- **THEN** 侧边栏背景为深 slate（#0f172a 附近），文字为浅灰/白色；内容区背景为近白（#f8fafc 附近）
- **AND** 激活项通过左侧强调色条与高亮背景标识

### Requirement: 工作台信息密度提升
The system SHALL 在工作台以更大的数字、更紧凑的列表、更少的装饰卡片展示关键信息。

#### Scenario: 首页概览
- **WHEN** 用户进入工作台
- **THEN** 顶部展示 2-3 个超大数字指标（如待处理工单、未读事件）
- **AND** 下方以两栏布局展示待办与动态，不使用四列等宽卡片

### Requirement: 表格专业感
The system SHALL 的表格看起来更像数据表格而非卡片列表。

#### Scenario: 数据列表
- **WHEN** 用户打开任意数据列表页
- **THEN** 表头与数据行有明显层级差异
- **AND** 行高统一、斑马纹清晰、hover 状态为整行背景变化
- **AND** 状态标签使用小圆角、低饱和背景

## MODIFIED Requirements

### Requirement: 页面头部
页面头部 SHALL 不再使用「渐变背景 + 左侧色条」的卡片式头部，而是使用简洁的标题 + 描述 + 可选操作按钮组合。

### Requirement: 统计卡片
统计卡片 SHALL 从圆角大卡片改为更克制的数字展示或窄条卡片，趋势信息使用小字号和箭头符号（↑/↓）而非图表。

### Requirement: 操作按钮
表格操作列 SHALL 使用 icon 按钮或简洁文字链接，不再使用彩色胶囊按钮。

## REMOVED Requirements

### Requirement: 经典商务蓝主题
**Reason**: 用户明确反馈该主题显得廉价且 AI 化
**Migration**: 使用新的强调色（赭石/琥珀/森林绿方向），主蓝色仅保留在信息链接或系统提示中

### Requirement: 圆角小卡片堆砌
**Reason**: 造成页面拥挤、同质化
**Migration**: 改用扁平分区、留白与细边框组织内容，圆角统一收敛到 4-6px
