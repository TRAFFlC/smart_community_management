### 🔍 代码审查报告

#### 严重问题（必须修复）

**1. [架构级] God Class — MainWindow 承担了过多职责**

- [MainWindow.h](file:///d:/smart_community_management/src/widgets/MainWindow.h) + 各 Page 文件合计超过 **7000+ 行**。`MainWindow` 作为 `QWidget` 子类，竟然承载了 Dashboard、Todo、Message、Archive、Property、Governance、Service、Report、System 共 **9 大类、40+ 子页面**的创建逻辑，是典型的 God Class。
- 每个页面创建函数都是数百行的巨型方法。例如 [createDashboardPage()](file:///d:/smart_community_management/src/pages/DashboardTodoMessagePage.cpp) 1059行、[createGovernancePage()](file:///d:/smart_community_management/src/pages/GovernancePage.cpp) 1655行、[createSystemPage()](file:///d:/smart_community_management/src/pages/SystemPage.cpp) 1934行。
- **修复方向**: 每个页面应独立为 `QWidget` 子类，MainWindow 只负责布局和路由：

```cpp
// 重构前：所有页面逻辑塞在 MainWindow 的成员函数里
QWidget* MainWindow::createSystemPage(const QString& sub) { /* 1934行... */ }

// 重构后：每个页面独立为一个类
class UserManagementPage : public QWidget { /* ... */ };
class RoleManagementPage : public QWidget { /* ... */ };
```

**2. [严重] 密码安全 — SHA256 + 硬编码盐值**

- [Utils.h#L13-L18](file:///d:/smart_community_management/src/utils/Utils.h#L13-L18): `hashPassword()` 使用 `SHA256(password + "smart_community_salt")`，这是**裸 SHA256 + 固定盐**，在 GPU 暴力破解面前形同虚设。
- **修复方向**: 至少使用 PBKDF2/bcrypt，每用户随机盐：

```cpp
// 使用 QCryptographicHash + 随机盐 + 多轮迭代
inline QString hashPassword(const QString& password) {
    QByteArray salt = QCryptographicHash::hash(
        QByteArray::number(QRandomGenerator::global()->generate()),
        QCryptographicHash::Sha256).toHex().left(16);
    QByteArray hash = QCryptographicHash::hash(
        (password + salt).toUtf8(), QCryptographicHash::Sha512);
    // 多轮迭代（至少 10000 轮）
    for (int i = 0; i < 10000; ++i)
        hash = QCryptographicHash::hash(hash, QCryptographicHash::Sha512);
    return salt.toHex() + ":" + hash.toHex();
}
```

**3. [严重] SQL 注入风险 — 表名/列名直接拼接**

- [DatabaseManager.h#L109](file:///d:/smart_community_management/src/database/DatabaseManager.h#L109): `insert()` 方法中表名和列名通过字符串拼接构造 SQL，虽然值使用了 `bindValue`，但列名来自 `QVariantMap` 的 key，如果上游传入恶意 key 可能导致注入。
- [DatabaseManager.h#L151](file:///d:/smart_community_management/src/database/DatabaseManager.h#L151): `count()` 方法中 `where` 参数直接拼接到 SQL 字符串中：

```cpp
QString sql = QString("SELECT COUNT(*) FROM %1 WHERE %2 AND del_flag = 0").arg(table, where);
```

- **修复方向**: 对表名和列名增加白名单校验。

**4. [严重] 资源泄漏 — MainWindow 中的原始指针管理混乱**

- [main.cpp#L78-L84](file:///d:/smart_community_management/src/main.cpp#L78-L84): `mainWindow = new MainWindow()` 后通过 lambda 捕获管理，logout 时 `delete mainWindow`，但 `m_pageCache` 中缓存的 40+ 页面 QWidget 指针从未被清理。

- **ProfileDialog 顶部渐变**: [ProfileDialog.cpp#L37-L38](file:///d:/smart_community_management/src/dialogs/ProfileDialog.cpp#L37-L38) 的 `qlineargradient(x1:0, y1:0, x2:1, y2:0, stop:0 #b45309, stop:1 #d97706)` 是全项目最精致的视觉细节。
